# Stock Market Simulator

A simplified stock exchange with a Python backend and React frontend.

---

## What's in this project

```
stockmarket/
├── backend/
│   ├── main.py          ← The entire API (all 7 endpoints, ~100 lines)
│   ├── requirements.txt ← Python packages to install
│   └── test_main.py     ← Automated tests (14 tests)
└── frontend/
    ├── src/
    │   ├── index.js     ← React entry point (3 lines, don't touch)
    │   └── App.js       ← The entire UI
    ├── public/
    │   └── index.html   ← HTML shell
    └── package.json     ← Node packages to install
```

## Step 1 — Install Python

**Windows**
1. Go to https://python.org/downloads
2. Click the big yellow "Download Python 3.x.x" button
3. Run the installer — **check the box "Add Python to PATH"** before clicking Install
4. Open a new Command Prompt and verify: `python --version`
   → should print something like `Python 3.12.3`

**macOS**
```bash
# Option A: from python.org (same as above)
# Option B: with Homebrew
brew install python
python3 --version
```

**Linux (Ubuntu/Debian)**
```bash
sudo apt update && sudo apt install python3 python3-pip -y
python3 --version
```

---

## Step 2 — Install Node.js

Node.js is needed to run the React frontend.

1. Go to https://nodejs.org
2. Download the **LTS** version (left button)
3. Install it (just click Next everywhere)
4. Verify in a new terminal: `node --version` → `v20.x.x`

## Step 3 — Download / clone this project

If you received a zip file: unzip it somewhere, e.g. `C:\Projects\stockmarket` or `~/projects/stockmarket`.

If you have Git:
```bash
git clone <your-repo-url>
cd stockmarket
```

---

## Step 4 — Start the backend

Open a terminal and run these commands one by one:

```bash
# Move into the backend folder
cd stockmarket/backend       # Windows: cd stockmarket\backend

# Install the required Python packages (only needed once)
pip install -r requirements.txt

# Start the server on port 8000
python main.py 8000
```

You should see:
```
✅ API running at http://localhost:8000
📖 API docs at  http://localhost:8000/docs
```

**Leave this terminal open.** The server runs as long as this window is open.

> **Port already in use?** Change 8000 to anything else, e.g. `python main.py 8001`

---

## Step 5 — Start the frontend

Open a **second terminal** (keep the backend terminal running):

```bash
# Move into the frontend folder
cd stockmarket/frontend      # Windows: cd stockmarket\frontend

# Install Node packages (only needed once, takes ~1 minute)
npm install

# Start the React development server
npm start
```

Your browser will open automatically at **http://localhost:3000**

---

## Step 6 — Use the app

The UI has five panels:

| Panel | What it does |
|-------|-------------|
| **Bank inventory** | Add stocks and quantities, then press "Save bank state" |
| **Trade** | Enter a wallet ID (any name) + stock name, click Buy or Sell |
| **Wallet lookup** | Type a wallet ID to see what it holds |
| **Audit log** | Shows every successful buy/sell in order |
| **Chaos engineering** | Kills the server process (restart it with `python main.py 8000`) |

**Quick demo — try this sequence:**
1. Bank panel → add `AAPL` qty `10`, add `GOOG` qty `5` → Save
2. Trade panel → wallet `alice`, stock `AAPL` → Buy 1 (do it 3 times)
3. Trade panel → wallet `bob`,   stock `GOOG` → Buy 1
4. Wallet panel → look up `alice` → see 3 × AAPL
5. Bank panel → AAPL quantity is now 7
6. Log panel → 4 entries in order

## API Reference (all 7 endpoints)

You can test these directly at **http://localhost:8000/docs** (interactive UI auto-generated by FastAPI).

| Method | Path | What it does |
|--------|------|-------------|
| `GET`  | `/stocks` | Get all stocks in the bank |
| `POST` | `/stocks` | Set bank inventory (replaces everything) |
| `POST` | `/wallets/{wallet_id}/stocks/{stock_name}` | Buy or sell 1 unit |
| `GET`  | `/wallets/{wallet_id}` | Get all holdings of a wallet |
| `GET`  | `/wallets/{wallet_id}/stocks/{stock_name}` | Get quantity of one stock |
| `GET`  | `/log` | Get full audit log |
| `POST` | `/chaos` | Kill this server instance |

### Example: set bank stocks
```bash
curl -X POST http://localhost:8000/stocks \
  -H "Content-Type: application/json" \
  -d '{"stocks": [{"name": "AAPL", "quantity": 100}]}'
```

### Example: buy a stock
```bash
curl -X POST http://localhost:8000/wallets/alice/stocks/AAPL \
  -H "Content-Type: application/json" \
  -d '{"type": "buy"}'
```

## Error codes

| Code | Meaning |
|------|---------|
| 200 | Success |
| 400 | Bad request (e.g. bank has no stock left, or wallet doesn't hold the stock) |
| 404 | Stock name doesn't exist in the bank |

## Common problems

**`pip` not found on Windows**
→ Re-run the Python installer, check "Add Python to PATH", then open a **new** terminal.

**`npm install` fails with network errors**
→ Check your internet connection. Corporate networks sometimes block npm; try a different network.

**Port 3000 already in use**
→ React will ask "use port 3001 instead? (y)" — just press Y.

**Frontend shows "Failed to fetch"**
→ Make sure the backend is running (`python main.py 8000`) and the port matches.

**`ModuleNotFoundError: No module named 'fastapi'`**
→ You're in the wrong folder, or forgot to run `pip install -r requirements.txt`.
